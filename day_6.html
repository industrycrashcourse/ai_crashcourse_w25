Two Approaches to AI-Assisted TDD

Human Tests → AI Code

  • Tests as specifications
  • Clear acceptance criteria
  • AI focuses on implementation
  • Human validates correctness

AI Tests ← Human Requirements

  • Requirements as prompts
  • AI suggests test cases
  • Human refines coverage
  • Implementation follows tests
